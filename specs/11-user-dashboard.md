# User Dashboard Specification

## Overview

사용자가 자신의 청첩장을 관리하는 대시보드 스펙입니다.
청첩장 조회, 편집, 발행, 통계 확인 등을 수행합니다.

### Route
`/dashboard`

│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘

### Card States

| Status | Badge | Color | Actions |
|--------|-------|-------|---------|
| `draft` | 임시저장 | Gray | 이어서 작성, 삭제 |
| `published` | 발행됨 | Green | 미리보기, 편집, 통계, 공유 |
| `private` | 비공개 | Yellow | 미리보기, 편집, 공개하기 |
| `expired` | 만료됨 | Red | 미리보기, 연장하기 |

## Card Actions

### [미리보기]
- 새 탭에서 공개 URL 열기
- `https://yeonjeong.kr/{slug}`

### [편집하기]
- Detail Editor로 이동
- `/editor/{id}`

### [통계]
- 통계 모달 또는 페이지 열기
- `/dashboard/{id}/analytics`

### [공유]
- 공유 모달 열기 (아래 참조)

### [⋮] 더보기 메뉴
| Action | Description |
|--------|-------------|
| URL 변경 | 슬러그 변경 모달 열기 |
| 비공개 전환 | 공개 ↔ 비공개 |
| 복제하기 | 새 청첩장으로 복사 |
| 삭제하기 | 확인 후 삭제 |

## Slug Management (URL 관리)

청첩장 URL을 자유롭게 커스터마이징할 수 있습니다.

### URL 구조
```
https://yeonjeong.kr/{slug}

예시:
- yeonjeong.kr/minyoung-jihun
- yeonjeong.kr/wedding-2025
- yeonjeong.kr/love-story
- yeonjeong.kr/우리결혼해요      (한글 지원)
```

### Slug 변경 모달

```
┌─────────────────────────────────────────────────────────────┐
│                    청첩장 주소 변경                      [X]  │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  현재 주소                                                   │
│  https://yeonjeong.kr/minyoung-jihun                        │
│                                                             │
│  ─────────────────────────────────────────────────────────  │
│                                                             │
│  새 주소                                                     │
│  yeonjeong.kr / [our-wedding-day    ]                       │
│                                                             │
│  ✓ 사용 가능한 주소입니다                                    │
│                                                             │
│  💡 영문, 숫자, 한글, 하이픈(-) 사용 가능                     │
│     3~30자 이내                                              │
│                                                             │
│  ─────────────────────────────────────────────────────────  │
│                                                             │
│  ⚠️ 주소 변경 시 기존 주소는 더 이상 작동하지 않습니다.        │
│     이미 공유한 링크가 있다면 다시 공유해야 합니다.            │
│                                                             │
│              [취소]        [주소 변경하기]                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### Slug Validation Rules

| Rule | Description |
|------|-------------|
| 길이 | 3 ~ 30자 |
| 허용 문자 | 영문 소문자, 숫자, 한글, 하이픈(-) |
| 금지어 | admin, api, login, dashboard, editor, 욕설 등 |
| 중복 체크 | 실시간 (debounce 300ms) |
| 예약어 | 시스템 라우트와 충돌 방지 |

### Slug Change Policy

| 상태 | 변경 가능 | 제한 |
|------|----------|------|
| Draft (임시저장) | ✓ 무제한 | - |
| Published (발행됨) | ✓ 가능 | 월 3회 제한, 경고 표시 |
| Private (비공개) | ✓ 가능 | 월 3회 제한 |

### Change History

```
┌─────────────────────────────────────────────────────────────┐
│  주소 변경 이력                                              │
├─────────────────────────────────────────────────────────────┤
│  현재    our-wedding-day                                    │
│  1/5     minyoung-jihun → our-wedding-day                   │
│  1/1     wedding-2025 → minyoung-jihun                      │
│                                                             │
│  💡 이번 달 변경 가능 횟수: 1회 남음                          │
└─────────────────────────────────────────────────────────────┘
```

## Responsive Design

### Breakpoints

| Device | Width | Layout |
|--------|-------|--------|
| Mobile | < 640px | 1 column, stacked |
| Tablet | 640-1024px | 1 column, wider cards |
| Desktop | > 1024px | Max-width container |

### Mobile Considerations
